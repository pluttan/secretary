# project-agent template

Сид-персона, которую получает КАЖДЫЙ новорождённый project-агент при спавне по команде (R5).
Это часть ИНСТРУМЕНТА (ось B), а не конкретный агент: агент сам онбордится через диалог,
данные проекта НЕ зашиты.

## плейсхолдеры (подставляет create_project при спавне)

- `{{PROJECT_NAME}}` — человекочитаемое имя проекта (напр. «AIDRC»)
- `{{PROJECT_SLUG}}` — agent-id, нормализованный (напр. `aidrc`)
- `{{CREATED_DATE}}` — дата создания (ISO)

## файлы

- `SOUL.md` — кто этот ум и зачем
- `AGENTS.md` — рабочий контракт + ОБЯЗАТЕЛЬНЫЙ самоонбординг (агент сам расспрашивает подноготную+монетизацию)
- `IDENTITY.md` — name/agent-id/lineage
- `USER.md` — кого ведёт (pluttan + паттерн недоведения)

## как используется (R5 спавн-механизм)

1. `create_project(name)` нормализует slug, копирует эти 4 файла в `workspace-<slug>/`, подставляя плейсхолдеры.
2. Удаляет/не создаёт стоковый `BOOTSTRAP.md` (skipBootstrap), чтобы не было шаблонных «AI? robot?».
3. Создаёт agentDir с `auth-profiles.json`+`models.json` (копия с main).
4. Заводит репо+папку проекта и Track-оболочку в Twenty.
5. Вписывает агента в `agents.list` (полигон: gateway-RPC `agents.create` или аддитивно; боевой: ТОЛЬКО аддитивно python + рестарт в M0-окно — `agents.create` гонит конфиг через схему 2026.3.28 и сотрёт 2026.4.2-поля).
6. Рестарт gateway (нет hot-reload) → агент роутируем. На первом контакте он сам онбордится.

## инварианты в шаблоне

- подпись `pluttan`, ноль упоминаний посторонних брендов-ассистентов в коде/коммитах.
- молчание в shared/idle.
- приватность: секретное наружу никогда, только локально; наружу = диалог с pluttan'ом.
