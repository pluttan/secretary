## что умеет инструмент (тулзы оркестратора)

Все доступны главному уму (Шики) как инструменты; вызываются в разговоре из телеги.

### рождение/смерть проекта
- **`create_project(name, [repoPath])`** — рождает выделенный ум проекта: репо+папка, Track-оболочка в Twenty (BACKLOG), workspace из шаблона `~/secretary/templates/project-agent/`, agentDir с ключами, запись в `agents.list`. Новый агент на первом контакте САМ расспрашивает подноготную+монетизацию. (Требует короткого рестарта gateway для активации — см. ниже.)
- **`kill_project` / движок `kill_project.py`** — убивает проект с эпитафией: пишет `~/secretary/graveyard/<slug>.md` (метаданные+причина+срез SOUL), Track→KILLED, workspace АРХИВИРУЕТСЯ (не стирается), агент убирается из `agents.list`.

### портфель (анти-распыление)
- **`wip_status`** — стадии всех проектов + WIP-вердикт (сколько ACTIVE, есть ли слот).
- **`set_project_stage(name, stage)`** — сменить стадию (BACKLOG/ACTIVE/FROZEN/SHIPPED/KILLED). **Активация ГЕЙТИТСЯ**: если активных уже на WIP-лимите (деф. 3, файл `~/secretary/spawn/wip.conf`) — откажет, «заморозь что-то прежде». Это лекарство от распыления.
- **`set_project_money(name, amount, [currency=RUB])`** — денежная цель.
- **`prioritize`** — советует, на чём фокус (из активных; источник истины по приоритетам — `## now` в STATE секретаря).

### синк и передача
- **`crm_bot_sync(project, [stage], [money], [note])`** — ОДИН вызов держит в синхроне Twenty Track и наг-лист бота `## now`: ACTIVE → проект в `## now` (M0 о нём напоминает), FROZEN/SHIPPED/KILLED → убирает. С откатом при рассинхроне.
- **`handoff_to_agent(agentId)`** / **`return_to_main()`** — передать разговор project-агенту (он отвечает в этом чате) и забрать обратно. Стоп-слова `/return`, `/unfocus`, «верни управление» — авто-возврат.

## как ты заводишь проект (поток)

1. В телеге: «заведи проект X» → Шики зовёт `create_project("X")`.
2. Инструмент рождает агента (репо+Track+workspace) и **рестартит gateway** (нет hot-reload — короткий блип, ~5с).
3. Пишешь новому агенту — он сам расспрашивает: что за проект, кому, как монетизируется, ближайшая цель. Запоминает в своей памяти.
4. Дальше ведёт проект: стадии/деньги командой (`set_project_stage`/`set_project_money`), фокус через `prioritize`, синк `crm_bot_sync`.
5. Закрыть → kill с эпитафией.

## рабочие нюансы (важно)

- **Рестарт для активации**: openclaw не умеет hot-reload агентов — после `create_project`/`kill_project` нужен рестарт gateway. На полигоне (`openclaw --dev`, порт 19001) — свободно. На БОЕВОМ (`openclaw.service`, 18789) рестарт = M0-чувствителен: только в окно без активных M0-пинков, не во флапе egress.
- **Боевой конфиг правим ТОЛЬКО аддитивно** (движки так и делают: json round-trip сохраняет 2026.4.2-поля; `agents add`/`plugins install` бинарём 2026.3.28 их сотрут — НЕ использовать).
- **M0 священен**: secretaryd + m0judge + наг-контур не ронять. Движки трогают `## now` секретаря — это влияет на якоря M0, так что тестовые токены туда не лить (для тестов `crm_sync --state-file <копия>`).
- **Приватность**: наружу только telegram тебе + Twenty; секретное (экран/OCR) судит локальный Qwen, наружу никогда.
